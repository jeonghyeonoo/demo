 구글 지도를 활용하여 목적지 저장.

4. 기술 스택

Java. 모바일 서비스로 확장 가능 및 python 보다 빠른 속도로 처리하는 JVM 엔진

JavaScript(Vanilla JS). 브라우저 호환성이 좋고 라이브러리를 사용할 때 보다 속도가 빠름

Google API. 전 세계 대상 서비스 및 가장 많이 사용되는 지도

Spring Boot H2. 간단한 demo DB. 서비스하게 되면 backend 로직 변경 없이 옮기기 가능 (PostgreSQL 또는 MySQL)

6. 설계 의도

-Book Print API를 활용 가능한 서비스를 생각한 결과 여행 앨범을 선정했다. 이후 서비스를 기획하면서 여행 전 과정을 함께하는 서비스를 떠올렸고 여행 계획을 세우고 여행 중 사진을 기록해 앨범 제작까지 할 수 있는 어플리케이션을 구현했다.

-기획한 것 중 추가되지 않은 기능이 있는데, 여행 계획을 공유할 수 있도록 한다면 더 많은 사용자를 확보할 수 있다. 한 가지 생각해보아야 할 점은 Google Maps API 사용에 대한 요금이 부과된다는 것이다.
(기본 제공되는 사용량을 초과하면 1000번 호출을 기준으로 요금이 부과된다. 서비스 특성 상 목적지 1개에 1번 호출한다고 하면 1일 동안 5개의 목적지(5번 호출)는 5일 여행하는 그룹 기준 25번의 호출. 계산하면 400 그룹 기준 $7이다.)

-공유 기능과 함께 비교적 간단한 로그인 기능을 추가하고 싶고 사소한 기능들(여행계획 카드 순서 바꾸기 등)을 계획했다.
